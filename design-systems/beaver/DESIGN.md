# Beaver UI

This is the **only** design system in the fork. Tokens and components are
not described in prose here — they are pulled directly from the live
`@beaver-ui/*` packages and the inner DS by `pnpm beaver:sync` and surfaced
to the LLM via:

- [`skills/beaver-prototype/components.json`](../../skills/beaver-prototype/components.json)
- [`skills/beaver-prototype/references/index.md`](../../skills/beaver-prototype/references/index.md)
- [`skills/beaver-prototype/references/tokens.md`](../../skills/beaver-prototype/references/tokens.md)

## Authority chain

1. The inner DS owns the tokens (`packages/design-tokens/*.ts` in its
   source repo).
2. Beaver consumes those tokens AND wraps / re-exports primitives of the
   inner DS as `@beaver-ui/*` packages.
3. This fork bundles both into a single UMD via
   `apps/beaver-runtime/`. The iframe preview loads it from
   `/vendor/beaver.umd.js` and exposes everything on `window.Beaver`.
4. The LLM is shown only the public surface (`@beaver-ui/*` plus the
   inner-DS primitives flagged as `tier: primitive`). It must never invent
   tokens, never invent components, never write raw HTML.

If you need to update the design system, do not edit this file. Instead:

```bash
pnpm beaver:refresh   # re-sync specs, then rebuild the runtime UMD
```
